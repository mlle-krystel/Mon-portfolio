Ohmyfood — Étude de cas HTML & CSS

Intégration mobile-first & animations CSS

Intégration front-end d’une application mobile-first de réservation de menus gastronomiques.

Ce projet met l’accent sur la structuration HTML, le CSS moderne et les animations d’interface, afin de proposer une expérience utilisateur fluide et élégante, sans recourir au JavaScript.

HTML Sass Animations CSS Mobile-first
Voir le résultat

Contexte

Ohmyfood est une startup fictive spécialisée dans la réservation de menus gastronomiques en ligne.

Le projet consistait à concevoir l’interface front-end d’une application mobile-first permettant aux utilisateurs de consulter des restaurants partenaires et de composer leur menu avant leur visite.

L’enjeu principal était de proposer une expérience utilisateur fluide et moderne, en mettant l’accent sur le design, les animations et la hiérarchie visuelle, tout en respectant des contraintes fortes de performance et d’accessibilité.

Ce projet avait également pour objectif de consolider les bases de l’intégration HTML/CSS, en travaillant exclusivement sans JavaScript, afin de comprendre jusqu’où il est possible d’aller avec du CSS moderne et bien structuré.

Objectifs

L’objectif du projet était d’intégrer une interface complète en approche mobile-first, fidèle aux maquettes fournies, tout en garantissant une navigation fluide et intuitive sur mobile, tablette et desktop.

Il s’agissait également de mettre en œuvre des animations CSS avancées afin d’enrichir l’expérience utilisateur, sans impacter les performances du site.

Le projet visait enfin à structurer le CSS de manière professionnelle à l’aide de Sass, afin de produire un code maintenable, évolutif et facilement lisible, tout en respectant les bonnes pratiques d’accessibilité et de responsive design.

Créer un site mobile-first Mettre en place des animations avancées en CSS Structurer des pages restaurant dynamiques Travailler la performance & accessibilité Garantir une UX fluide et intuitive

Réalisations principales

Le projet a été réalisé en intégration HTML et CSS, avec une architecture Sass en 7-1 permettant de séparer clairement les styles globaux, les composants, les layouts et les animations.

L’interface a été pensée en mobile-first, avec une adaptation progressive pour les écrans plus larges, garantissant une lisibilité optimale et une hiérarchie claire des contenus.

Plusieurs animations CSS ont été implémentées, notamment sur les cartes de restaurants, les boutons, les loaders et les éléments interactifs, afin de rendre l’expérience plus dynamique sans recourir à JavaScript.

Une attention particulière a été portée aux performances et à l’accessibilité, en limitant les effets coûteux, en structurant correctement le HTML et en assurant des contrastes lisibles et une navigation fluide.

Design mobile-first complet Animations : boutons, cartes, loaders Menus structurés par sections Effets d’apparition progressifs Sass : architecture 7-1

Résultats

Avec Ohmyfood, j’ai développé une interface responsive mobile-first, en suivant les principes d’accessibilité (navigation clavier, contrastes, lisibilité). Ce projet m’a permis d’expérimenter les animations CSS pour enrichir l’expérience utilisateur sans JavaScript. J’ai structuré mon SCSS avec une architecture modulaire et utilisé des variables pour faciliter les déclinaisons de thèmes. Le site final est fluide, accessible, et l’intégration fidèle à la maquette Figma renforce sa qualité perçue sur tous les écrans.

UX moderne & engageante Animations fluides Interface claire et accessible
Recul sur le projet

Ce projet m’a permis de concevoir une interface en partant de fortes contraintes techniques, notamment l’absence de JavaScript. Avec le recul, je vois comment ce travail pourrait être approfondi, par exemple en renforçant la cohérence des animations et la hiérarchie visuelle afin d’améliorer encore la lisibilité et le confort d’utilisation.
