Kasa — Étude de cas React & architecture SPA

Application de location immobilière avec routage dynamique

Refonte complète du front-end en React avec une architecture SPA moderne. Mise en place d’un routing dynamique, de composants réutilisables et d’une gestion des erreurs, le tout intégré fidèlement à partir de maquettes Figma.

React React Router SCSS SPA Vite
Voir le résultat

Contexte

Kasa est une entreprise de location immobilière entre particuliers souhaitant moderniser son site web existant.

L’objectif du projet était de refondre entièrement le front-end sous forme de Single Page Application afin d’améliorer la maintenabilité du code, la navigation et l’expérience utilisateur.

Les données des logements sont fournies via un fichier JSON local, simulant une source de données externe. Le site devait être construit à partir de maquettes Figma, avec un respect strict du design et un comportement responsive sur tous les supports.

L’enjeu principal était de structurer l’application autour de composants réutilisables, d’un routing dynamique et d’une gestion propre des erreurs et des redirections.

Objectifs

L’objectif de ce projet était de concevoir une Single Page Application complète avec React, en adoptant une architecture claire, modulaire et maintenable, proche des standards utilisés en environnement professionnel.

Le site devait permettre l’affichage dynamique des logements à partir de données fournies, avec une navigation fluide entre les pages grâce à un système de routing performant, incluant la gestion des routes dynamiques et des erreurs de navigation.

Ce projet avait également pour but de consolider les bases de React, notamment la création et l’organisation de composants réutilisables, la gestion des props et la structuration globale d’une application front-end moderne.

Enfin, une attention particulière devait être portée au respect fidèle des maquettes Figma, à l’accessibilité de l’interface et à l’adaptation responsive sur mobile, tablette et desktop, afin de garantir une expérience utilisateur cohérente sur l’ensemble du parcours.

Créer une SPA complète en React Implémenter un routing dynamique Construire des composants réutilisables Développer une fiche logement complète Gérer les erreurs et redirections

Réalisations principales

Le projet a été initialisé avec Vite afin de bénéficier d’un environnement de développement rapide et moderne.

J’ai conçu l’architecture de l’application en séparant clairement les pages, les composants, les layouts et les données, afin de garantir une meilleure lisibilité et évolutivité du code.

Le routing dynamique a été mis en place avec React Router, incluant la gestion des paramètres d’URL et une page 404 pour les routes inexistantes.

Plusieurs composants réutilisables ont été développés, tels que les cartes de logements, la galerie d’images, les collapses et le système de notation, permettant une interface cohérente et maintenable.

L’ensemble de l’application respecte les maquettes Figma et s’adapte aux différents formats d’écran grâce à une approche responsive.

Initialisation du projet avec Vite Architecture pages, composants, data, layout Routing dynamique + page 404 Composants clés : Card, Gallery, Collapse, Rating… Responsive fidèle aux maquettes Figma

Résultats

Le projet Kasa m’a permis de construire une application en React de A à Z, à partir de maquettes Figma. J’ai structuré l’interface en composants réutilisables (Card, Gallery, Collapse…) et mis en place un système de routing dynamique avec React Router. L’architecture claire et modulaire rend le code facilement maintenable. Résultat : une Single Page Application fluide, responsive, conforme aux maquettes et prête à se connecter à une vraie API. L’expérience utilisateur est également renforcée par la gestion des routes invalides et une page 404 dédiée.

Application scalable & maintenable Navigation fluide & robuste UX moderne & cohérente
Recul sur le projet

Ce projet m’a permis de comprendre l’importance de penser l’architecture d’une application React avant le développement des composants. Avec le recul, je vois comment cette structure pourrait évoluer dans un contexte plus proche de la production, notamment avec une gestion plus centralisée des données et des états.
